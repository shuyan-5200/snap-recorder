# 更新记录

## 0.2.0

- 新增可选“人声（麦克风）”，默认关闭，仅在用户主动开启时申请权限。
- 未录人声时保持原有一键自动保存；录了人声后可以单选或同时选择“完整视频”与“视频 + 人声分轨”。
- 系统声与人声使用独立音轨和统一时间轴；暂停区间会同时从画面、电脑声音和人声中移除。
- 合并导出只重新混合音频，H.264 视频码流原样透传，不降低画质。
- 高清 H.264 参数会先检查当前系统编码器是否支持；旧版 macOS 会安全回退，避免启动录制时崩溃。
- 分开导出会补齐人声首尾静音，使 M4A 与视频时长误差控制在 40 毫秒内。
- 人声为 AAC-LC 48 kHz / 192 kbps；合并音频为 48 kHz / 256 kbps。
- 新增等待导出时的退出保护、最多 3 个文件的访达定位、统一重名保护、麦克风权限说明与失败重试。
- 产品正式更名为 Snap Recorder，公开仓库准备为 `snap-recorder`，应用标识为 `io.github.shuyan-5200.SnapRecorder`。
- 增加开源 README、隐私说明、贡献指南、MIT License 与 GitHub Actions 自检。

## 0.1.4

- 浏览器与整屏模式改为原生像素优先，最高支持 3840×2160；当前 Retina 浏览器不再预先缩到 1440 高。
- 浏览器内容框改为整数像素对齐，禁止为填补舍入误差做轻微上采样，提升网页文字和细线清晰度。
- H.264 质量预算提升到约 24–68 Mbps，启用最高采集质量、CABAC 和编码质量优先。
- 高清录制前的剩余空间保护由 2 GB 提升到 5 GB。

## 0.1.3

- 浏览器成片不再固定为 16:9，改为跟随开始录制时的浏览器窗口比例，避免视觉上横向偏扁。
- 浏览器输出最高约 2K，窗口约占画布 95% 宽高，只保留一圈轻微壁纸背景。
- 缩小阴影范围，并补充比例、边距、偶数像素和整屏尺寸回归自检。

## 0.1.2

- 修复关闭主窗口后再次双击 Snap Recorder，却没有重新显示窗口的问题。
- 统一为单一安装入口，避免多份 Snap Recorder 同时运行造成入口与权限混乱。

## 0.1.1

- 修复可见浏览器窗口被旧 `isOnScreen` 状态误判为最小化、导致无法开始录制的问题。
- Snap Recorder 回到前台时自动刷新浏览器窗口；已选窗口关闭后要求重新选择，不会静默切换目标。
- 正式进入录制状态前等待首个完整画面；无画面时约 3 秒内返回明确提示。
- 停止采集后才重新显示 Snap Recorder，避免整屏成片末帧混入菜单栏状态。
- 缓存浏览器壁纸与阴影合成，降低 1080p 录制时的实时处理压力。
- “系统声音”改为“电脑声音（不含麦克风）”，减少理解歧义。

## 0.1.0

- 首个可运行 MVP：浏览器窗口 / 整个主屏幕、3 秒倒计时、暂停继续、自动导出 MP4。

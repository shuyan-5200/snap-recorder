为何选择

只做录屏该做的事

没有剪辑器，没有云同步，没有账号体系。打开、录制、保存——路径最短，画质最高。

原生像素优先

最高 3840×2160，H.264 High Profile。合并人声时视频轨不二次编码，画质不打折。

声音分别可控

电脑声音与人声独立开关。需要时一次导出完整成片，或把人声分轨留给剪辑。

全程本地处理

无账号、无统计、无网络请求。录屏与声音只在本机完成，文件直接进「下载」。

界面不进成片

倒计时、主窗口与控制胶囊自动隐藏。你录到的，只有你想留下的画面。

小而完整

Universal App 约 2.2 MB，零第三方依赖。SwiftUI + ScreenCaptureKit 原生实现。

最短路径

授权 → 选来源 → 3 秒倒计时 → 录制 → 自动保存。不需要设置格式，也不用选文件夹。

两种模式

浏览器，或整个屏幕

按场景选来源。浏览器不必最大化，只要没有最小化即可录制。

浏览器

只录选中的浏览器窗口，自动铺满画面并保留少量桌面背景。其他应用与窗口不会进入成片。

整个屏幕

录制当前主显示器，保持原始比例与原生像素。适合演示、教学与完整工作流记录。

声音与导出

使用流程

四步完成一次录屏

从授权到成片，路径被刻意压到最短。

  1. 授权

    首次启动允许屏幕与系统音频；需要人声时再开麦克风。

  2. 选择来源

    浏览器或整个屏幕，按需打开电脑声音与人声。

  3. 录制

    3 秒倒计时后开始。可随时暂停、继续或结束。

  4. 自动保存

    文件写入「下载」。可选在访达中显示，或再录一个。

# Snap Recorder 项目规则

- 产品目标是极简录屏，不新增编辑器、剪辑、变焦、云分享等重功能。
- 默认中文界面，流程必须保持“选来源 → 倒计时 → 录制 → 自动保存”。
- 浏览器模式必须坚持独立窗口采集后再合成背景，不能退化成屏幕区域裁切。
- 整屏模式必须排除 Snap Recorder 自身应用；倒计时和录制 HUD 必须保持不可共享。
- 输出固定保存到下载目录，不加入保存面板。
- 人声默认关闭；未开启时必须保持“结束后自动保存一个 MP4”的原有流程。
- 开启人声后只提供“完整视频 / 视频与人声分轨”两张可多选导出卡，不增加格式或设备选择器。
- 合并人声时不得重新编码视频轨；分开导出不得打 ZIP。
- 实际代码修改后至少运行 `swift build -c release` 与 `.build/release/SnapRecorder --self-test`。
- 首次涉及真实屏幕或麦克风采集的验证，需要用户在 macOS 系统提示中批准对应权限。
- 完整 Xcode 不是必需条件；可以使用 SwiftPM 与 `scripts/build-app.sh` 构建 `.app`。
